# Indice de Issues - Scaffold API

Este documento centraliza todas as tasks e bugs do projeto. Cada entrada deve ter um documento detalhado em `docs/task/`.

---

## Como Usar

1. **Antes de comecar qualquer tarefa:** Crie uma entrada neste indice
2. **Documente o plano:** Crie `docs/task/TASK-###-slug.md` ou `docs/task/BUG-###-slug.md`
3. **Atualize o status:** Mantenha o status atualizado conforme avanca
4. **Ao concluir:** Crie `docs/implementations/IMPL-###-slug.md` com detalhes da implementacao

---

## Status

- `todo` - Nao iniciado
- `doing` - Em andamento
- `done` - Concluido

## Notas

- **Numeracao sequencial por tipo:** TASK-001, TASK-002, BUG-001, BUG-002, etc.
- **Data:** Formato YYYY-MM-DD
- **Slug:** Use kebab-case (ex: `criar-sistema-documentacao`)
- **Commits:** Referencie o ID (ex: `feat(docs): TASK-001 criar sistema de documentacao`)
